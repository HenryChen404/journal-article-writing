# Article Architecture

Use this reference to choose and control the whole-article structure.

## 1. Select the dominant line

### Logical progression

Best for explanatory and investigative features.

Common order:

```text
present development
→ scale
→ cause or mechanism
→ concrete impact
→ counterevidence or complication
→ reaction
→ future
```

### Chronology

Best when action itself carries meaning.

Possible spines:

- one day;
- one case;
- one journey;
- one product launch;
- one investigation;
- one historical sequence.

Background should enter only when the current action creates a need for it.

### Thematic profile

Best for a person, team, institution, or community.

Choose 2–3 demonstrated traits or tensions, such as:

```text
ambition vs caution
public confidence vs private doubt
technical mastery vs organizational dependence
```

Build each theme from action, not adjectives.

## 2. Standard feature architecture

### Lead

Create interest and embody or announce the central development.

### Nut graf

Explain the story's scale, stakes, and reporting promise.

### Section 1: establish the mechanism

Show what changed and how it works.

### Section 2: make the impact concrete

Move from aggregate fact to person, place, organization, or scene.

### Section 3: complicate

Introduce limits, counterevidence, tradeoffs, or an actor whose experience does not fit the simple story.

### Section 4: reaction

Show what people and institutions do because of the impact.

### Ending

Complete the central movement rather than adding a new thesis.

## 3. Section cards

Before drafting, create a card for each section:

```text
Section title:
Purpose:
Question answered:
Primary claim:
Best evidence:
Human/action element:
Counterpoint:
Transition out:
What is deliberately excluded:
```

A section without a unique purpose probably belongs elsewhere or should be deleted.

## 4. The evidence rhythm

Avoid long runs of the same material type.

Weak:

```text
expert quote
→ expert quote
→ expert quote
→ abstract summary
```

Stronger:

```text
claim
→ decisive number
→ participant action
→ short explanation
→ document
→ complication
```

Variation is not decoration. It prevents one evidence type from carrying a job it cannot do.

## 5. Long-form design

For pieces above roughly 2,500 words, add navigation:

- descriptive subheads;
- recurring character or question;
- clear time/place markers;
- deliberate returns to the main line;
- short summaries after complex sections;
- visible escalation or deepening.

Use controlled digressions only when they:

- explain a necessary mechanism;
- deepen character;
- reveal a causal history;
- set up a later turn.

Return to the main line with a concrete action or consequence.

## 6. Outline audit

Before drafting, ask:

- Can the article's movement be described in one sentence?
- Does each section change the reader's understanding?
- Is background placed where it becomes necessary?
- Are the strongest scenes and evidence distributed rather than clustered?
- Is the central conflict visible more than once?
- Does the complication genuinely challenge the easy version?
- Does the ending arise from the reported action?
- Could any section be removed without changing the argument? If yes, remove or merge it.

## 7. Alternative architectures

### Case-led explanation

```text
case scene
→ nut graf
→ mechanism
→ scale
→ second case or counterexample
→ institutional reaction
→ ending
```

### Investigation

```text
finding
→ how the evidence was obtained
→ affected people
→ responsible system or decision
→ response and rebuttal
→ broader pattern
→ unresolved accountability
```

### Reported essay

```text
specific observation
→ question
→ reported exploration
→ competing explanations
→ revised understanding
→ earned reflection
```

The reported essay may include the writer, but the writer's feelings are not evidence for external claims.

### Profile

```text
revealing action
→ why this person matters now
→ formative history
→ present conflict
→ contradictory trait
→ consequential decision
→ open future or echo
```
